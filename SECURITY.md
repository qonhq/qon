# Security Policy

## Supported Versions

Qon follows a rolling support model while in `v0.x`.

| Version | Supported |
| --- | --- |
| main | Yes |
| v0.x latest minor | Yes |
| older v0.x | Best effort |

## Reporting a Vulnerability

Please do not report security vulnerabilities in public issues.

1. Email maintainers at `security@qon.sh` (replace if needed).
2. Include a clear description, impact, and reproduction steps.
3. Include logs, PoCs, and affected versions when possible.

You can expect an acknowledgement within 72 hours.

## Disclosure Process

- We confirm and triage the report.
- We prepare and validate a fix.
- We coordinate disclosure timing with the reporter.
- We publish a security advisory and patched release notes.
